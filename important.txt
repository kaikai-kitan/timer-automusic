1. システム要件（System Requirements）
1.1 機能要件（実装すべき機能）
タイマー機能:

ユーザーが任意の時間（分・秒）を設定できる機能。

よく使われる時間（例: 5分、25分ポモドーロなど）のプリセットボタン。

スタート、一時停止、リセット機能。

タイマー終了時の通知（アラーム音や画面上のポップアップ）。

BGM自動取得・再生機能:

設定されたタイマー時間に基づいて、YouTubeから適切な長さの動画（作業用BGM、Lo-Fi、自然音など）を自動検索する機能。

タイマーのスタートと同時に、取得したYouTube動画の音声をバックグラウンド（または埋め込みプレイヤー）で自動再生する機能。

BGMの音量調整、ミュート機能。

取得したBGMが気に入らない場合、別のBGMにスキップ（再検索）する機能。

1.2 非機能要件
レスポンシブ対応: PC、タブレット、スマートフォンで快適に動作するUI。

パフォーマンス: タイマーの遅延がないこと。API通信時のローディング状態を適切にユーザーにフィードバックすること。

エラーハンドリング:

設定した時間と完全に一致する動画が見つからない場合のフォールバック処理（例: 設定時間より長い動画を取得し、タイマー終了時に再生を停止する）。

APIのレート制限（利用制限）に達した場合の代替処理。

1.3 推奨技術スタック
モダンでパフォーマンスが高く、開発体験の良い構成をご提案します。

フロントエンド: Svelte (SvelteKit)

軽量で状態管理がしやすく、1秒ごとにUIを更新するタイマーアプリの要件に非常に適しています。

ホスティング / サーバーレス: Vercel

SvelteKitとの相性が抜群で、APIキーを隠蔽するためのサーバーレス関数のデプロイも容易です。

バックエンド / データベース: Supabase

初期リリース（MVP）では不要かもしれませんが、将来的に「ユーザーのお気に入りBGM履歴」や「カスタムプリセットの保存」などを実装する際のBaaSとして最適です。

外部API:

YouTube Data API v3: キーワード（例: 作業用BGM {設定時間}分）で動画を検索・取得するため。

YouTube IFrame Player API: Webアプリ内でYouTube動画の再生、停止、音量調整などをJavaScriptから制御するため。

2. 設計要件（Design Requirements）
2.1 UI/UX設計
ミニマルな構成: 主役はあくまで「タイマー」と「作業への集中」であるため、複雑な検索画面は設けず、時間を入力してスタートボタンを押すだけのシンプルな導線にします。

プレイヤーの配置: 映像よりも音声がメインとなるため、YouTubeのIFrameプレイヤーは画面の隅に小さく配置するか、デザインに溶け込むようなUI（動画のサムネイルを背景にぼかして配置するなど）を検討します。

2.2 システムアーキテクチャ（処理の流れ）
ユーザー入力: クライアント（ブラウザ）でユーザーがタイマーの時間を設定し、Startを押す。

APIリクエスト: SvelteKitのサーバーサイド（エンドポイント）へ、設定時間を含めたリクエストを送信。

YouTube検索: サーバーサイドからYouTube Data APIを叩き、q="作業用BGM 5分" などのクエリで検索し、Video IDを取得（※フロントエンドから直接叩くとAPIキーが漏洩するため、必ずサーバーサイドを経由します）。

クライアントへの返却と再生: 取得したVideo IDをフロントエンドに返し、YouTube IFrame Player APIに渡して動画をロード。タイマーのカウントダウンと同時に再生を開始。

終了処理: タイマーが0になったら、IFrame Player APIを通じて動画を自動停止させる。
